What happens when you initialize a repository? Why do you need to do it?

	You create an .git directry that stores your commits. In order to
	start using the git's features.

How is the staging area different from the working directory and the repository?
What value do you think it offers?

	The staging area is the place where you add the files to be committed.
	The working directory is the place where you make the changes and create
	the commits.
	The repository is the place where you have the code files.

How can you use the staging area to make sure you have one commit per logical
change?

	Because you can add the files that you need to change for the logical
	change.

What are some situations when branches would be helpful in keeping your history
organized? How would branches help?

	Situations such as: having language version of a program, different
	features of a program and so on. Branches let you have these different
	versions of a program and you can work on them independently.

How do the diagrams help you visualize the branch structure?

	Because you can see which commits belong to which branches and vice
	versa.

What is the result of merging two branches together? Why do we represent it 
in the diagram the way we do?

	The result is a new branch. Because in the end, both commits become
	one commit. And that commit will be a branch.
